## 📝 Steps to Activate Your Profile

### Step 1: Create the Profile Repository

1. Go to GitHub: https://github.com/new
2. **Repository name MUST be exactly:** `ajaybe-ops` (same as your username)
3. Make it **Public**
4. **DO NOT** initialize with README, .gitignore, or license (we already have these)
5. Click "Create repository"

### Step 2: Push the Files

Once the repository is created, run these commands in the `ajaybe-ops` directory:

```bash
cd ../ajaybe-ops
git remote set-url origin https://YOUR_TOKEN@github.com/ajaybe-ops/ajaybe-ops.git
git push -u origin main
```

**Note:** Replace `YOUR_TOKEN` with your GitHub Personal Access Token, or use GitHub CLI (`gh auth login`) for authentication.

### Step 3: Enable GitHub Actions (Optional but Recommended)

1. Go to your repository: https://github.com/ajaybe-ops/ajaybe-ops
2. Go to **Settings** → **Actions** → **General**
3. Under "Workflow permissions", select: **"Read and write permissions"**
4. Check: **"Allow GitHub Actions to create and approve pull requests"**
5. Save

This will enable the automatic metrics update workflow that runs daily.

## 🔧 Troubleshooting

**Profile not showing?**
- Make sure the repository is named exactly `ajaybe-ops` (same as username)
- Make sure it's **Public**
- Make sure the branch is named `main` or `master`
- Make sure README.md is in the root directory

**Metrics not updating?**
- Go to repository → Actions tab
- Check if the workflow ran successfully
- Enable GitHub Actions in repository settings if needed

**Stats not showing?**
- The stats APIs (github-readme-stats) are external services
- They may take a few minutes to update after first push
- Check your username spelling in all URLs
